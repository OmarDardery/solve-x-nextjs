# SolveX Branding Specification (Authoritative)

This document defines the official branding rules for SolveX.
AI agents and automation tools MUST follow this specification exactly
when modifying or applying branding to any website or interface.

This file is derived directly from the official SolveX branding PDF.

---

## 1. Brand Concept

SolveX branding is based on:

> Combining two opened and skimming book icons to form an **X**.
>  
> The brand represents the **infinite search for knowledge, development,
> and sustainability**.

All visual decisions must support this concept.

---

## 2. Logo Assets

All logo files are located in the **public directory**.

### Available Logo Files

| File Name        | Description |
|------------------|-------------|
| `logo.png`       | Standard logo |
| `BlackLogo.png`  | Black logo variant |
| `WhiteLogo.png`  | White logo variant |
| `Icon&Type.png`  | Icon combined with logotype |

### Logo Rules

- Logos must not be stretched, rotated, recolored, or modified
- Maintain original proportions at all times
- Ensure sufficient empty space around logos
- Logos must remain sharp and readable at all sizes

---

## 3. Color Palette (Official)

The following colors are the **only approved brand colors**.

### Primary Brand Colors

| Hex Code |
|---------|
| `#011c8d` |
| `#1329b3` |
| `#643ae6` |
| `#a15df2` |

### Color Usage Rules

- Colors must be used exactly as defined (no opacity changes unless required by UI state)
- Gradients may only be created using the above colors
- Do not introduce additional brand colors
- Maintain sufficient contrast for accessibility

---

## 4. Typography

### Official Typeface

**Source Sans (Variable)**

### Font Characteristics

- Clean
- Neutral
- Works in almost any context
- Suitable for technical and editorial content

### Language Support

- English font: **Source Sans**

### Typography Rules

- Avoid ALL CAPS text
- Use **sentence case** for:
  - Headlines
  - Subheadings
  - Body text
- ALL CAPS allowed **only when required**, such as:
  - Brand collaborations
  - Poor print readability scenarios

---

## 5. Text Alignment & Layout

### Alignment Rules

- All text must be **left-aligned**
- Layout must leave enough space for the stylized **X icon**
- Additional text must:
  - Be smaller than the main text
  - Appear **under** the main text when paired

These rules must be respected in branding, media content, and UI layouts.

---

## 6. Headlines & Subheadings

- Use sentence case
- Avoid decorative capitalization
- Maintain visual clarity and hierarchy
- Ensure consistency across all pages and components

---

## 7. Accessibility & Clarity

AI agents must ensure:

- Adequate contrast between text and background colors
- Logos remain visible and readable
- Branding does not reduce usability or legibility

---

## 8. Forbidden Actions

The following are NOT allowed:

- Changing brand colors
- Introducing new fonts
- Using non-left-aligned text for branded content
- Applying effects (glow, shadow, blur) to logos
- Mixing unapproved visual styles

---

## 9. Enforcement Rule (For AI Agents)

If a branding decision is not explicitly defined in this document:

1. Do NOT invent or infer new rules
2. Preserve existing branded elements
3. Default to minimal, neutral UI changes
4. Maintain brand integrity over aesthetics
